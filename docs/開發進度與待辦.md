## 第二批回饋修正（已寫好、未 commit、需重新部署 v12 + 實測）
9.  **分表匯入範圍限定回寫**（修「從甲階段匯入會誤刪其他階段分表」）：
    分表匯入後不再全量 `_do_export`+`_do_split`，改呼叫 `_do_sync_category`
    → gs action **`syncCategory`**：只(a)取代總表中該類別的列(其他類別保留)、
    (b)只重建該類別分表。gs 端 `syncCategory()`+`replaceCategoryRowsInMain()`
    (讀不到表頭就不動總表避免誤刪)。py 端 `_build_export_records_for_category`、
    `plan["category"]`。
    - 測：從某階段分表改值/加列→匯入→確認只有該階段分表/總表該階段列被更新，
      其他階段分表沒被動到。
10. **分表固定參數版面**：名稱放「圖紙號碼」欄上方、值放「圖紙名稱」欄上方
    (這兩欄不會被隱藏)，名稱/值各佔一格。`writeSubSheet` 用 labels.indexOf 定位；
    `readSheet` 從「圖紙名稱」欄讀固定參數值。產生或覆蓋更新分表都套用。
    - 測：匯出拆分或分表匯入後，看分表上方版面是否正確。
11. **新增/編輯圖紙後詢問是否同步**（純 Revit 端，不需重新部署）：
    `main()` 重構成迴圈 + `_open_manager_window(initial_tab)`。create/edit 成功後跳
    `_prompt_sync_after_change()`(是/否)：選「是」→重開管理器切到 `MODE_SYNC` 分頁；
    選「否」→只更新 Revit。build_env/import_apply 處理完直接結束。
    - 測：管理器新增/編輯圖紙→確認跳詢問、選「是」會跳到同步分頁。
12. **匯入前 UID 異常檢查**（純 Revit 端，不需重新部署）：分表/總表常見資料錯誤
    ——「有 UID 但無值(空白列)」「重複 UID(UID 被往下填)」會造成建垃圾圖或誤刪。
    `_on_import_preview` 在 dup-number 檢查前，先 `_ImportCleanupWindow.has_issues`
    判斷，有問題就跳 `_ImportCleanupWindow`：條列異常列，勾選＝該列不匯入(移除)，
    每個 UID 最多保留 1 列(驗證)，某 UID 一列都不留→對應圖紙會被刪(後面刪除確認再列)。
    保留列放 `cleaned_records` 續跑。`_build_uid_sheet_map` 提供對應圖號圖名顯示。
    背景：空白列因 UID 非空仍會被 readSheet 當資料 → 對應不到圖會建空白垃圾圖；
    重複 UID 會讓多列指向同一圖、其他圖被判定移除而誤刪。
    - 測：分表故意留有 UID 的空白列、或把 UID 往下填造成重複 → 匯入應跳檢查視窗。

## 同步資料流速記
- 匯出 records 含 7 欄(UID/是否為Revit出圖/圖紙類別/圖紙號碼/圖紙名稱/繪圖員/修正備註)
  + 5 個固定參數(供拆分填標頭)。
- 匯入 readSheet 回傳 records/role/category/batchParams/mainTab。
  role=sub → Revit 端 `_build_plan(scope_category, batch)` 鎖類別 + 固定參數併入。
- 套用在 `_apply_plan`(交易在 main() 內，關窗後才執行)；預覽用 `_plan_to_rows`。
- gs actions：`read` / (預設)寫總表 / `split` 拆全部分表 / `syncCategory` 範圍限定。
- 角色：`setRole/getRole`(DeveloperMetadata BAF_ROLE = main/sub)。
