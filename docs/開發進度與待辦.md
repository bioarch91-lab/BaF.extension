## Google Sheet 端「BaF」自訂選單（純選單版，預設權限即可）
- ⚠ **不要手動列 appsscript.json 的 oauthScopes**(維持預設讓 Google 自動偵測)。
  之前手動列權限→連 `script.container.ui` 都要自己補、頻頻卡授權。純選單功能
  (SpreadsheetApp/HtmlService/Ui)用自動偵測就好。
- ⚠ **不做「雲端自動更新腳本」**(Apps Script API 要 GCP 啟用+授權，太麻煩，已放棄)。
  腳本由使用者**自己貼上**：擴充功能→Apps Script→貼上覆蓋→存檔→部署新版本。
- `onOpen()` 建立「BaF」選單，兩個功能，**一律鎖 UID 比對**；下拉只列
  「同時有 UID／圖紙名稱／圖紙號碼」的工作表(`bafQualifies_`/`bafListSyncSheets`)；
  **總表由使用者自己從下拉選**(不靠 role，允許副本備份產生多個總表)。
  清單由**伺服器端直接渲染進 HTML**(`bafBuildDialogHtml_`)，不靠瀏覽器端非同步取清單
  → 解決「選不了/空白」。`bafSelfTest` 可在編輯器 Run 看合格清單。
- **下拉/勾選依角色嚴格分開**：分表勾選區只列 role=sub、總表下拉只列 role=main
  (`bafBuildDialogHtml_` 用 `list.filter`)。沒 main 時下拉提示先用 Revit 匯出建總表。
- **手動設定角色**(選單「設定目前工作表角色」→ `bafMarkMain`/`bafMarkSub`/`bafClearRole`
  → `bafSetActiveRole_`)：複製總表當存檔備份時，副本的 BAF_ROLE 中繼資料不一定會跟著
  複製，用這個把副本也設成 main，它就會出現在「總表」下拉(允許多個總表並存)。
  ⚠ 多個 main 時 Revit 端 `findMainTabName` 會取「工作表順序第一個 main」→ 建議把存檔
  副本放在現役總表「之後」，避免 Revit 自動回寫寫到備份。
- **列的種類 `bafRowKind_`**(關鍵規則)：
  - 有ID沒資料(圖號圖名皆空)=`delete` 刪除標記 → **同時刪總表與分表**。
  - 沒ID有資料=`new` 新增(匯入 Revit 後才配 UID)。
  - 有ID有資料=`normal`(依 UID 比對修改)；沒ID沒資料=`junk` 忽略。
- **兩段式：預覽差異 → 確認執行**(`disabled` 的執行鈕，預覽後才開放)：
  - ①「分表 → 更新總表」`bafBuildSubsToMain_`：分表來源分類成 delete/normal/new；
    總表列：在 delSet→刪、本身是刪除標記→刪、UID 對到 normal→修改、
    在分表類別內但分表已無(`scopeCats`)→刪、其餘保留；normal/new 未對到→新增。
    apply 後也把被刪 UID/標記列從各分表移除(刪兩邊)。
  - ②「總表 → 更新分表」`bafBuildMainToSubsPlan_`：分表＝總表該類別、非刪除標記的列。
    總表刪除標記→從總表與分表都移除(`mainClean` 回寫總表)。
    分表 UID 在總表類別→修改、總表多出→新增、分表多出/被標記→刪除。
- **安全(req3)**：`bafFindDupUids_` 偵測結果重複 UID → 預覽⛔並擋執行、apply throw；
  刪除一定列在預覽(紅字，不會誤刪)；修改/刪除/新增因 `bafRowKind_` 分類正確而不再漏讀。
- 本機已驗證 4 項：角色過濾、subs→main(改1刪2新1)、main→sub(改1新1刪1)、
  主表刪除標記連動刪分表。
- **加速(req1)**：`bafFastWrite_` 總表用「一次清+一次 setValues 整塊」批次寫(欄位連續時)，
  不連續才退回逐欄 writeIndexToSheet；分表沿用 writeSubSheet。
- 本機已驗證：差異邏輯(add/mod/del)、重複 UID 偵測、對話框 HTML 與內嵌 JS 語法皆過。
- doPost(Revit 同步)維持 v12，改 .gs 後仍要重新部署 /exec 才生效；
  選單功能存檔+重新整理試算表即生效。
